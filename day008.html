DAY 008 · 2026-06-17

我想给AI装个"插件"，
结果被公司电脑拦了

本来想"晋级"一下，给AI接上更大的能力。
撞了一次墙，反而想明白一件事：我根本不需要它。

🔌 踩坑 + 反转
第一章

我听说AI能装"插件"，挺兴奋

前七天，我都是跟AI聊天、让它帮我写东西、做小工具。今天想学个"高级"点的——给AI装插件（正式名字叫 MCP 连接器）。

简单说，插件能让AI从"只会说"变成"能动手操作真实的工具"。我想先装一个 GitHub 插件，让AI以后能直接管我的网站仓库，不用我每次手动操作。

📖 名词解释

插件 / MCP 连接器：相当于给AI接上一只"手"，让它能伸进某个外部工具里去操作——比如你的 GitHub、Gmail、笔记软件。装上插件，AI就不只是"建议你怎么做"，而是"直接帮你做"。

第二章

一行命令，当场撞墙

AI给了我一行命令，我满怀期待地敲下回车——结果红字报错：

Windows PowerShell
PS> claude mcp add --transport http github "https://api.githubcopilot.com/mcp/" ...
✗ Cannot add MCP server "github": not allowed by enterprise policy
🚧
翻译成人话：公司不让装
我这台是公司发的电脑，公司IT用"企业策略"禁止了员工自己装插件。这是公司的安全规定，我改不了，也不该去绕过它。所以——在工作电脑上，装插件这条路是堵死的。

说实话，第一反应有点泄气：搞了半天，白学了？

第三章

等一下……我好像根本不需要它

冷静下来我想：我装这个 GitHub 插件，是为了让AI能管我的仓库。可是——

昨天 Day007 那篇，不就是AI直接帮我推上线的吗？全程一个插件都没用啊。

对啊。AI早就能管我的 GitHub 了。因为 GitHub 自己提供了一个"遥控器"——一个叫 gh 的命令行工具，它已经装在我电脑上、也登录好了。AI直接拿这个遥控器用就行，压根不需要额外的插件。

第四章

这下我才真懂了：插件到底是干嘛的

插件不是"越多越高级"。它有一个非常具体的用途：

插件，是用来连
"AI够不着的东西"的。
够得着的，根本不用插件

把AI想象成一个很能干的助理：他只能用手边够得着的工具。GitHub 这个工具正好在他手边（gh命令），拿起来就用；而 Gmail、笔记软件那些锁在"别的房间"里的工具，才需要插件这把钥匙去开门。

第五章

这篇文章，就是证据

说一千道一万，不如直接证明给你看。

你现在正在读的这篇 Day008，从写出来到发布上线，全程是AI用 gh 命令帮我搞定的，没有用任何插件。我只说了一句"把今天的经历做成Day008，发出去"。

✅ 你能看到这页 = 证明成立

这页能出现在你面前，就说明：AI不需要装插件，也能直接管我的 GitHub 仓库、把网站更新上线。我想要的那个"能力"，其实昨天就已经在手里了。

🧠 今天最大的收获

新手老有一个错觉：以为要装个"高级功能"才算厉害。但很多时候，你手里的基础工具已经够强了，只是你没意识到。够得着的东西，别绕远路去装钥匙。

我以为我缺一个插件。其实我缺的，是看清自己手里已经有什么。

📅 接下来

→ 插件这东西很有意思，等哪天用自己的私人电脑，再装个玩玩（比如连Gmail、连笔记）
→ 继续推进那个真正想做的东西：让AI帮我审查代码的小工具
→ 记住今天这个教训：先看清手里有什么，再决定要不要去找新工具
