第一章

早上打开微信，什么都没有

昨天 Day 005 设置好两个定时任务，一个每天早上8点发半导体日报，一个盘中每5分钟监控铠侠股价。满以为今天早上微信会有通知，结果什么都没有。

今早的对话
我
你赶紧看看，昨天晚上我让你做股票预测，早上应该8点和9点以后给我发邮件，怎么一封邮件我都没有收到啊？
🤖
先查一下定时任务的状态……找到两个问题：第一，Python命令写的是 python3，但你的 Windows 上只有 python；第二，Gmail 的 App Password 失效了，认证报535错误。
我
密码失效这个问题很大啊。为什么非要登录我自己的邮箱？能不能换一种方式？
🤖
可以用 Server酱（方糖）发微信，不用登录你的邮箱，只需要一个 Token，永久有效，不受账号登录行为影响。
第二章

一次诊断，发现了两个 bug

AI 查到了定时任务的状态，发现两个独立的问题同时存在——难怪一封邮件都没收到。

🐛
Bug 1：python3 在 Windows 上不存在
SKILL.md 里写的是 python3，但这台 Windows 电脑只有 python。任务每次触发就静默失败，根本没有执行到发邮件那一步。
✅ 修复：把两个 SKILL.md 里的 python3 改成 python
🔑
Bug 2：Gmail App Password 失效
Gmail 的应用专用密码在你从其他设备登录、或 Google 安全检测触发时会自动撤销。这不是操作错误，是系统机制——你无法控制。
✅ 修复：彻底抛弃 Gmail SMTP，换成微信推送
第三章

换掉邮件，改用微信推送

Server酱（方糖）是一个专门做微信推送的服务。扫码登录，拿到一个 SendKey，以后发通知就一行代码搞定，Token 永久有效。

🔍 关键洞察

邮件要登录你的账号才能发出去，所以账号的任何变动都会影响它。而 API 通知（Server酱、Telegram、企业微信等）是第三方帮你传递消息，只需要一个独立 Token——跟你的个人账号完全解耦。以后做任何自动通知，首选 API，不要 SMTP。

第四章

但还有一个致命问题：关电脑就挂了

微信通知解决了，但用户问了一个更根本的问题：

「那今天是 Day006，就研究这个吧。以后主要是用 AI 做的工作，就是让它 24 小时不间断工作，电脑不开机也能工作才行。」

这才是 AI Agent 的终极状态——不依赖任何人的电脑，在云端自己运转。

解决方案：GitHub Actions。GitHub 给每个仓库提供免费的云端服务器，可以按时间表自动执行脚本，每月 2000 分钟免费——对我们的用量来说完全够用，而且是真正的云端，关机没影响。
