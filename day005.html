问题：等我看到新闻已经晚了

日本股市9点开盘，但真正影响开盘价的是前一天晚上美国的走势。等我早上起来打开手机，翻完新闻，最佳操作时机往往已经过了。

我持仓两只股票：铠侠控股（285A）20股、英伟达（NVDA）5股。半导体行业受美国政策影响极大——出口管制、关税、AI芯片需求，随便一条新闻都能让股价大幅波动。

03

关键设计：预警阈值怎么定？

铠侠（285A）是2024年底刚上市的公司，业务100%依赖NAND闪存价格，AI需求拉动下波动极大。查了历史数据：

历史极值参考
2026年2月12日财报发布：单日 +12.36%
2026年3月31日美国景气忧虑：单日 -11.85%
年初来股价区间：¥11,350 → ¥24,420（约115%振幅）
正常交易日波动估算：±2~3%

基于此，设计了三级预警体系：

🔴 ≥ ±7% 紧急预警 — "立即查看！" 明显异常，可能有重大消息
🟡 ±4%~7% 注意预警 — "建议关注" 高于正常波动1.5倍以上
🟢 < ±4% 正常范围 — 不发邮件，静默继续监控

05

实操过程

整个搭建过程大概花了30分钟，三个关键步骤：

第一步：Gmail App密码 — 不能用普通密码，要在Google账号里生成专用的16位"应用密码"，只能发邮件，随时可撤销，安全。

第二步：测试Gmail SMTP连通 — 用PowerShell发了一封测试邮件，秒到。

# PowerShell 测试 Gmail SMTP
$cred = New-Object PSCredential("gmail地址", $securePass)
Send-MailMessage -SmtpServer smtp.gmail.com -Port 587 -UseSsl ✓

第三步：创建两个定时任务 — 在Claude Code的"ルーチン"（定时任务）里，分别设置了每天8点和每5分钟的自动触发。

Python smtplib Gmail SMTP Yahoo Finance API PowerShell Claude Scheduled Tasks WebSearch WebFetch

中途发现一个错误：把铠侠的股票代码写成了6600，实际应该是 285A。好在AI帮我找到了所有需要改的地方，一起修正了。

今天最大的感悟
以前我以为"自动化"就是定时执行脚本，写死逻辑。
今天做的是：AI自己判断新闻重不重要，自己决定邮件内容，自己选择要不要发。

这才是Agent和普通脚本的区别——有判断力，不只是执行力。
